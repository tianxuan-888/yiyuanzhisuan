# 会员端功能更新

## 新增功能

### 1. 收益端口展示
在会员端首页顶部添加了**总收益**统计卡片，显示用户的历史累计收益。

**展示位置**：会员端首页 → 统计卡片区域（第4个卡片）

**显示内容**：
- 图标：📈 TrendingUp（上升趋势）
- 标签：总收益
- 数值：累计获得的所有收益（单位：元）
- 样式：紫色渐变背景

### 2. 充值能量值功能
在能量值卡片上添加了**充值按钮**，点击后弹出充值对话框。

**充值流程**：
1. 点击能量值卡片右上角的"充值"按钮
2. 选择充值金额（50/100/500/1000）或手动输入
3. 填写备注信息（可选）
4. 提交充值申请
5. 系统自动发送通知给绑定的服务商
6. 会员联系服务商线下付款
7. 服务商确认后在系统内充值

**充值对话框功能**：
- 显示充值说明（能量值用途、充值流程）
- 快捷金额选择按钮（50/100/500/1000）
- 自定义金额输入
- 备注信息输入
- 提交申请按钮

**充值申请接口**：`POST /api/member/energy-recharge`

**请求参数**：
```json
{
  "userId": "member-001",
  "amount": 100,
  "note": "备注信息"
}
```

**响应示例**：
```json
{
  "success": true,
  "message": "充值申请已提交，请联系 服务商A 线下付款",
  "data": {
    "providerName": "服务商A",
    "amount": 100
  }
}
```

## 技术实现

### 前端修改
**文件**: `src/app/member/page.tsx`

1. **新增状态变量**：
   - `showRechargeDialog`: 控制充值对话框显示
   - `rechargeAmount`: 充值金额
   - `rechargeNote`: 充值备注

2. **新增处理函数**：
   - `handleRechargeEnergy`: 处理充值申请提交

3. **UI修改**：
   - 能量值卡片：添加"充值"按钮
   - 总收益卡片：将"总持仓"改为"总收益"，显示 `stats.total_profit`
   - 充值对话框：完整的充值表单UI

### 后端接口
**文件**: `src/app/api/member/energy-recharge/route.ts`

**接口功能**：
- 验证用户信息
- 获取绑定的服务商信息
- 创建充值通知（发送给服务商）
- 返回充值申请结果

**业务逻辑**：
1. 查询用户信息
2. 获取用户绑定的服务商
3. 创建通知记录（发送给服务商）
4. 返回服务商联系方式信息

## 使用说明

### 充值能量值步骤

1. **登录会员账号**
   - 访问：`/member`
   - 使用会员账号登录（如：member1 / member123）

2. **进入充值对话框**
   - 点击能量值卡片右上角的"充值"按钮

3. **填写充值信息**
   - 选择或输入充值金额（最小50）
   - 填写备注信息（可选）
   - 点击"提交申请"

4. **联系服务商付款**
   - 系统会显示服务商名称
   - 联系服务商线下付款

5. **等待充值到账**
   - 服务商收到款项后，在系统中充值
   - 刷新页面查看能量值变化

### 查看收益

1. 登录会员账号
2. 查看首页顶部的"总收益"卡片
3. 显示累计获得的所有收益金额

## 数据结构

### 统计卡片数据
```typescript
interface Stats {
  energy_value: number;    // 能量值
  balance: number;         // 余额
  points: number;          // 积分
  total_holding: number;   // 总持仓（已移除）
  total_profit: number;    // 总收益（新增）
  product_count: number;   // 持有产品数量
}
```

```typescript
  related_id: null

```

## 注意事项

1. **充值金额限制**：建议最小充值金额为50能量值
2. **线下付款**：充值后必须联系服务商线下付款
3. **服务商标题**：用户必须绑定服务商才能充值
4. **通知机制**：充值申请会自动发送通知给绑定的服务商
