# 纪元智科 - 三类数据关系梳理

## 概述

本项目有**三类核心数据**，它们相互关联，共同支撑整个业务运转：

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           第一类：用户账号体系                                │
│                    (谁是谁，有什么样的上下级关系)                               │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                          第二类：算力额度流转                                 │
│                    (额度从哪来，怎么分配，怎么变成产品)                          │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                          第三类：能量值流转                                   │
│              (能量值从哪来，怎么流通，最后怎么消耗/销毁)                         │
└─────────────────────────────────────────────────────────────────────────────┘
```

---

## 第一类：用户账号体系

### 核心表：`users`

| 字段 | 说明 | 示例 |
|------|------|------|
| `id` | 用户唯一标识 | `uuid` |
| `username` | 用户名 | `branch1` |
| `phone` | 手机号 | `13800000002` |
| `role` | 角色 | `admin` / `branch` / `provider` / `member` |
| `provider_id` | 所属服务商ID | `provider-001` |
| `branch_id` | 所属分公司ID | `branch-id-xxx` |
| `inviter_id` | 推荐人ID | `user-id-xxx` |

### 用户关系树

```
总公司 (admin)
    │
    └── 分公司 (branch)
            │
            └── 服务商 (provider)
                    │
                    └── 会员 (member)
```

### 角色说明

| 角色 | 说明 | 主要功能 |
|------|------|----------|
| `admin` | 总公司 | 创建额度、分配给分公司、查看全局 |
| `branch` | 分公司 | 分配额度给服务商、审核服务商申请 |
| `provider` | 服务商 | 生成产品、管理会员、充值能量值 |
| `member` | 会员 | 购买产品、卖出变现、申请服务商 |

## 第二类：算力额度流转

### 核心表

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `company_quota` | 总公司额度总账 | `total_quota`, `used_quota`, `available_quota` |
| `quota_allocations` | 额度分配记录 | `branch_id`, `provider_id`, `quota_amount`, `used_amount` |
| `providers` | 服务商额度账户 | `user_id`, `quota`, `used_quota` |
| `products` | 产品表 | `provider_id`, `price`, `period`, `status` |
| `user_products` | 用户持仓表 | `user_id`, `product_id`, `purchase_price`, `status` |

### 额度流转路径

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  第一步：总公司生成额度                                                        │
│  company_quota.total_quota += 额度                                           │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│  第二步：总公司分配给分公司                                                     │
│  quota_allocations: branch_id = 分公司ID, provider_id = null                 │
│  company_quota.used_quota += 分配额度                                         │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│  第三步：分公司分配给服务商                                                     │
│  quota_allocations: branch_id = 分公司ID, provider_id = 服务商ID              │
│  providers.quota += 分配额度                                                   │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│  第四步：服务商生成产品                                                        │
│  products: provider_id = 服务商ID, price = 产品价格                           │
│  providers.used_quota += 产品价格                                              │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│  第五步：会员购买产品                                                          │
│  user_products: user_id = 会员ID, product_id = 产品ID                        │
│  orders: order_type = 'buy', status = 'completed'                           │
└─────────────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│  第六步：产品到期/会员卖出                                                     │
│  orders: order_type = 'sell', status = 'pending'                           │
│  → 服务商审核通过 → 会员获得本金+收益                                           │
│  → 会员支付能量值（market_fee）                                               │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 第三类：能量值流转

### 核心表

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `energy_accounts` | 能量值账户 | `user_id`, `balance`, `total_in`, `total_out` |
| `energy_transactions` | 能量值流水 | `type`, `amount`, `from_user_id`, `to_user_id` |
| `energy_withdraw_requests` | 变现申请 | `amount`, `actual_amount`, `fee`, `status` |

### 能量值类型 (`energy_transactions.type`)

| 类型 | 说明 | 流转方向 |
|------|------|----------|
| `create` | 总公司创建能量值 | 系统 → 总公司 |
| `quota_match` | 额度匹配下发 | 总公司 → 分公司 |
| `purchase` | 分公司购买能量值 | 分公司 → 总公司 |
| `transfer_in` | 转入（市场内流转） | 服务商 → 会员 |
| `transfer_out` | 转出（市场内流转） | 会员 → 服务商 |
| `withdraw_freeze` | 变现冻结 | 账户 → 冻结池 |
| `withdraw` | 变现发放 | 冻结池 → 用户（扣除手续费） |
| `burn` | 能量值销毁 | 账户 → 系统（减少总供给） |
| `manual` | 手动调整 | 系统 ↔ 用户 |

### 能量值来源

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              能量值来源                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  1. 产品收益换算                                                              │
│     ├─ 会员卖出产品时支付 market_fee（能量值）                                  │
│     ├─ 分配比例：服务商70% / 公司5% / 直推10% / 上级服务商10% / 分公司5%        │
│     └─ 例：会员卖出产品，支付500能量值                                          │
│         → 服务商获得：500 × 70% = 350                                        │
│         → 公司运营：500 × 5% = 25                                            │
│         → 直推奖励：500 × 10% = 50                                           │
│         → 上级服务商：500 × 10% = 50                                         │
│         → 分公司：500 × 5% = 25                                              │
│                                                                             │
│  2. 总公司下发                                                               │
│     ├─ quota_match：总公司按额度比例给分公司下发能量值                           │
│     └─ 能量值作为市场流通的"燃料"，驱动产品流转                                  │
│                                                                             │
│  3. 分公司向总公司购买                                                        │
│     ├─ purchase：分公司购买能量值（线下打款给总公司）                            │
│     └─ 用于分公司能量值储备或下发给服务商                                       │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 能量值消耗

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              能量值消耗                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  1. 会员购买产品                                                              │
│     └─ 会员必须有足够能量值才能购买产品（产品成本的一部分）                        │
│                                                                             │
│  2. 变现提现                                                                 │
│     ├─ withdraw_freeze：申请变现时冻结能量值                                   │
│     ├─ withdraw：实际发放（扣除5%手续费）                                      │
│     │   └─ 实际到账 = 申请金额 × 95%                                          │
│     └─ burn：销毁能量值（与withdraw同时记录）                                 │
│         └─ 销毁金额 = 申请变现时的总能量值                                      │
│                                                                             │
│  3. 市场流转                                                                │
│     ├─ transfer_out：会员转给服务商                                            │
│     └─ transfer_in：服务商转给会员                                            │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 变现流程详解

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              变现流程                                       │
└─────────────────────────────────────────────────────────────────────────────┘

  1. 会员申请变现
     ┌──────────────────────────────────────────┐
     │ energy_transactions                     │
     │ type: 'withdraw_freeze'                 │
     │ amount: 1000 (冻结)                      │
     │ from_user_id: 会员ID                     │
     │ to_user_id: null                         │
     └──────────────────────────────────────────┘

  2. 总公司审核通过
     ┌──────────────────────────────────────────┐
     │ energy_withdraw_requests                 │
     │ status: 'approved'                        │
     │ amount: 1000 (申请金额)                    │
     │ fee: 50 (手续费5%)                        │
     │ actual_amount: 950 (实际到账)              │
     └──────────────────────────────────────────┘

     ┌──────────────────────────────────────────┐
     │ energy_transactions (两条记录)            │
     │                                          │
     │ ① type: 'withdraw'                       │
     │   amount: 950                            │
     │   → 发放给会员                           │
     │                                          │
     │ ② type: 'burn'                           │
     │   amount: 1000                           │
     │   → 销毁总能量值                         │
     └──────────────────────────────────────────┘

  3. 手续费沉淀
     ┌──────────────────────────────────────────┐
     │ 申请金额: 1000                            │
     │ 实际到账: 950                              │
     │ 手续费沉淀: 50 (5%)                       │
     └──────────────────────────────────────────┘
```

## 关键业务规则

### 1. 能量值分配比例（产品卖出时）

| 角色 | 比例 | 说明 |
|------|------|------|
| 服务商 | 70% | 主要收益来源 |
| 公司运营 | 5% | 公司留存 |
| 直推奖励 | 10% | 推荐人奖励 |
| 上级服务商 | 10% | 上级服务商奖励 |
| 分公司 | 5% | 分公司奖励 |

### 2. 变现手续费

| 项目 | 比例 |
|------|------|
| 手续费 | 5% |
| 实际到账 | 申请金额 × 95% |
| 销毁金额 | 申请金额（系统回收） |

### 3. 额度流转规则

| 流转方向 | 说明 |
|----------|------|
| 总公司 → 分公司 | 按配置比例分配，可选赠送能量值 |
| 分公司 → 服务商 | 按100%分配 |
| 服务商 → 产品 | 额度转换为产品 |
